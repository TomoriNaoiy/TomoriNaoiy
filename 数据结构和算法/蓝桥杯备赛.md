```python
n=int(input())
nums=list(map(int,input().split()))
ans=0
for i in nums:
  if i & (i - 1) == 0:
    ans += 1
print(ans)
```

# 第三题

<img width="824" height="926" alt="image" src="https://github.com/user-attachments/assets/49fe0cc9-9c06-43f2-8c7b-9c8de6343f20" />
很恶心的一题 用到的知识点很多 应该说是i很不错的一题

首先环形字符串最大共长 正常我们用动态规划来做 但是这里是环形 因此我们选择使用两倍原字符串 然后截取长度小于l的部分 就是环形匹配了 我们使用哈希表记录所有长度为l的部分 然后匹配的时候从大到小 如果切割长度中有跟这个长度相同的 所有最大共长就这么长 

确定边权后 观察到 这题是连接n个点用n-1边 是最大生成树类型 因此用克鲁斯卡尔算法 并且用并查集判断 就能完成

```python
import os
import sys

# 增加递归深度，防止并查集 find 递归过深报错
sys.setrecursionlimit(200000)

n, m = map(int, input().split())
p = []
for _ in range(n):
    p.append(input())

# 存放所有字符串的子串字典
all_subs = []
for s in p:
    ss = s + s
    # 字典初始化
    subs = {L: set() for L in range(1, m + 1)}
    for L in range(1, m + 1):  # 统一使用大写 L
        for i in range(m):
            subs[L].add(ss[i : i+L])
    all_subs.append(subs)

edges = []
for i in range(n):
    for j in range(i + 1, n):
        weight = 0
        # 从最大可能长度 m 开始倒序尝试匹配
        for L in range(m, 0, -1):
            # 利用 Python 集合求交集的特性匹配
            if all_subs[i][L] & all_subs[j][L]:
                weight = L
                break
        # 注意这里的缩进：在 for L 循环结束后，再将权值加入边集
        edges.append((weight, i, j))

# 倒序排序，参数是 reverse=True
edges.sort(key=lambda x: x[0], reverse=True)

parent = list(range(n))

def find(i):
    if parent[i] == i:
        return i
    # 路径压缩
    parent[i] = find(parent[i])
    return parent[i]

def union(i, v):
    pi = find(i)
    pv = find(v)
    if pi == pv:
        return False
    else:
        # 【关键修复】将 i 的根节点指向 v 的根节点
        parent[pi] = pv  
    return True

ans = 0
edges_added = 0
    
# 贪心选择边
for w, u, v in edges:
    if union(u, v):
        ans += w
        edges_added += 1
        # 当加够 n-1 条边时，树已生成，可以提前退出
        if edges_added == n - 1:
            break  
            
print(ans)
```

import math

n=int(input())
nums=[True for _ in range(100000)]
nums[0]=nums[1]=False 
for i in range(2,int(math.sqrt(100000))+1):
    if nums[i]:
        for j in range(i*i,100000,i):
            nums[j]=False
temp=[]
for i in range(2, 100000):
  if nums[i]:
    temp.append(i)
is_win = [False] * 100001

# 第五题
<img width="769" height="1161" alt="image" src="https://github.com/user-attachments/assets/b716a96e-6edb-49d5-bcb1-5e8b6eca5db3" />
这一题是比较有巧思的 和力扣的《最大数一样》 都是给出一些数字 然后组合成最大的数 十进制和二进制都有一个共同点 有不同的位数做为取舍 比如1和11 或者10和100这种难以取舍的 这时我们使用functools里面的cmp_to_key

这个函数的作用就是 在sort的时候 可以自己定下规则 如果1 就是a<b -1 就是a>b 0 就是按长度区分

因为我们只需要两两对比 然后发现哪一个顺序更大 就按这个顺序来排 这样最后排出来的就是最大值

```python
import os
import sys
from functools import cmp_to_key
# 请在此输入您的代码
n=int(input())
nums=[bin(i)[2:] for i in range(1,n+1)]
def compare(a,b):
  if a+b>b+a:
    return 1
  elif a+b<b+a:
    return -1
  else:
    return 0
nums.sort(key=cmp_to_key(compare))
nums=nums[::-1]
print(int(''.join(nums),2))

```

根据题目中“最小值的可能最大值” 我们使用二分答案 因为这是一个可以搜索可能性的问题 如果强行迭代来倒水的话 就会有一大堆的细节问题

因此
```python
import os
import sys

# 请在此输入您的代码
n,m=map(int,input().split())
nums=list(map(int,input().split()))
nn=[[] for _ in range(m)]
for i in range(n):
  nn[i%m].append(nums[i])
def check(ans):
  for i in nn:
    ext=0
    for j in i:
      if j+ext<ans:
        return False
      ext=j+ext-ans
  return True

low=0
high=100000000
aa=0
while low<=high:
  mid=(low+high)//2
  if check(mid):
    aa=mid
    low=mid+1
  else:
    high=mid-1
print(aa)

```
先分组 对于每一组 我们遍历一次 看这一轮的答案会不会小 不够大 就二分找 直到找到最大的答案

### 什么时候用二分答案呢
1. 字眼明显：题目结尾经常问**“最小化最大值”（比如：让耗时最长的那个人耗时尽可能短）、“最大化最小值”（比如：刚才那道倒水题，让水最少的瓶子里的水尽可能多），或者求某个特定的“最大/小可能值”**。
2. 答案存在一个明确的分界线。比如求“最大可行水量”：如果水量 $X$ 是可行的，那比 $X$ 小的水量肯定也都可行；
3. 正向求解极其困难：你发现从已知条件一步步推导最终答案，情况太多，完全无从下手。具备“单调性”（最核心）：如果 $X$ 不可行，那比 $X$ 大的水量肯定也都不可行。这就形成了一半是 True，一半是 False 的单调区间。
