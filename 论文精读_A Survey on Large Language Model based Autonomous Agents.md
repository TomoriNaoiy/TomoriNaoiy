# 统一框架
Profile（画像）： 决定 Agent 是谁（比如是程序员、老师还是某个特定性格的人）。

Memory（记忆）： 决定 Agent 能记住什么（历史经验和环境感知）。

Planning（规划）： 决定 Agent 如何思考（未来的行动计划）。

Action（行动）： 决定 Agent 如何与环境交互（执行代码、调用 API 等）。

2. 深入解析：Profiling（画像模块）

  1. 手工设定
  2. llm直接设置
  3. 数据集分析 通过大量数据统计来设定
3. 深入解析：Memory（记忆模块）
  1. 统一记忆（短期记忆）： 直接把信息塞进 Prompt 里，利用 LLM 的上下文窗口（In-context learning）来实现。优点是简单，缺点是受限于模型上下文长度，容易遗忘。

  2. 混合记忆（长短期结合）： 短期记忆处理当前上下文，长期记忆用于存储海量历史经验
记忆的操作（Memory Operations）：

这也是 Agent 能像人一样思考的核心机制。记忆提取（Memory Reading）： 面对当前任务，Agent 该回想起哪些记忆？论文总结了一个非常经典的提取打分公式：$$m^* = \arg\max_{m \in M} (\alpha s_{\text{rec}}(q, m) + \beta s_{\text{rel}}(q, m) + \gamma s_{\text{imp}}(m))$$$s_{\text{rec}}$ (Recency / 最近性)： 越近发生的事情得分越高。$s_{\text{rel}}$ (Relevance / 相关性)： 记忆与当前查询（Query）的向量相似度越高，得分越高。$s_{\text{imp}}$ (Importance / 重要性)： 这条记忆本身的权重（比如“着火了”比“吃早饭”重要得多）。记忆写入（Memory Writing）： 解决记忆重复（去重或融合）和记忆溢出（固定缓存，先进先出）的问题。记忆反思（Memory Reflection）： 斯坦福虚拟小镇（Generative Agent）的核心机制。Agent 会定期回顾底层的零碎记忆（比如：A在看书，A在和图书管理员聊天），然后利用大模型总结出高维度的抽象洞察（结论：A最近在专注于学术研究），从而指导未来的行为。

4. Planning（规划模块）
  1. 无反馈规划 (Planning without Feedback) 
    1. 单路径： 任务被分解为一条线性的、瀑布式的步骤（Step 1 -> Step 2 -> Step 3）。每一步只有唯一确定的后续步骤。 看起来效果应该并不好
    2. 多路径推理 (Multi-path Reasoning)
        核心逻辑： 类似人类面临选择时的思考。推理过程呈现树状（Tree）或图状（Graph）结构。在每一步，Agent 会构思出多个可能的下一步，然后进行评估和选择。
    3. 外部规划器 (External Planner)
        LLM 擅长“高层战略”，但“底层战术”（比如具体的低级代码执行或精确的数学搜索）容易出错。因此，把任务翻译成特定的形式化语言（如 PDDL），然后扔给传统的、极其精确的外部搜索算法（比如传统 AI 里的启发式搜索器）去计算路径，算完再让 LLM 翻译回人话。
  2.有反馈规划
    1. 环境反馈 (Environmental Feedback)
        核心逻辑： 采取行动后，观察虚拟环境或客观世界给出的结果（例如：游戏里的报错信息、搜索引擎的返回结果）。
    2. 人类反馈 (Human Feedback)
        核心逻辑： 在关键节点主动询问人类的意见。这能有效对齐人类价值观并减少模型的“幻觉”。
    .3. 模型内部反馈 (Model Feedback)
        核心逻辑： “左脚踩右脚”的自我进化。利用 LLM（可以是同一个，也可以是另一个充当“裁判”的 LLM）来评估刚刚执行的动作，并生成反思。 有点自监督学习的意思了

 # 评估体系：怎么证明你的 Agent 比别人的强？ 
 方案 A：主观评估 (Subjective Evaluation)
最直观但也最昂贵的方法。

人类标注 (Human Annotation)： 直接让人类来打分。

图灵测试 (Turing Test)： 让人类分辨不出这是机器干的还是人干的。

案 B：客观评估 (Objective Evaluation)
用冷冰冰的数据和榜单说话。

评估指标 (Metrics)： 任务成功率（Success rate）、人类相似度（Human similarity）、运行效率和成本（Efficiency）。

评估协议 (Protocols)： 是放在真实世界模拟器里跑（比如跑 Minecraft 游戏），还是放在社交环境里测试它的情商（Social evaluation），亦或是跑多任务测试集？

跑分榜单 (Benchmarks)： 学术界已经建好了很多考场。比如测试综合能力的 AgentBench，测试社交能力的 SocKET，测试工具调用能力的 ToolBench，以及测试网页操作的 WebShop。

# challenge
我认为或许对我们来说是更有价值的部分
. 角色扮演的瓶颈 (Role-playing Capability)
大模型虽然能演好“影评人”、“翻译官”，但如果让它扮演冷门职业或具有特定心理认知缺陷的人，它往往会演砸（缺乏自我意识）。目前只能靠微调（Fine-tuning）或写极度复杂的提示词，但很容易顾此失彼。

2. 泛化人类对齐的反直觉问题 (Generalized Human Alignment)
这绝对是个能发好文章的视角！传统的对齐是为了让模型“做个好人”（比如绝不能教人做炸弹）。但在社会模拟中，如果你想研究犯罪率或信息传播，Agent 就必须能够模拟坏人或拥有错误价值观的人。如何让目前被“阉割”得很乖的 ChatGPT/GPT-4 重新释放这些负面特征以供学术研究，是一个巨大的挑战。

3. 提示词的极度脆弱 (Prompt Robustness)
给 Agent 装上记忆和规划模块后，底层的 Prompt 会变得无比庞大和复杂。牵一发而动全身，改错一个标点符号可能整个 Agent 体系就崩溃了。目前大家只能靠“炼丹”式的手动试错，或者让大模型自己生成 Prompt。

4. 致命的幻觉 (Hallucination)
大模型最大的毛病就是“一本正经地胡说八道”。在普通的聊天里还好，但在 Agent 执行代码或操作真实系统时，幻觉会导致极其严重的安全隐患（比如删库跑路）。目前主流的解法是引入“人类在环（Human-in-the-loop）”来纠错。

5. 失控的“知识边界” (Knowledge Boundary)
RAG 是为了给模型外挂知识，但 Agent 模拟社会时，反而要限制模型的知识。大模型拥有“上帝视角”（比如它看过所有电影的剧情），但一个普通的虚拟人类 Agent 是不可能全知全能的。如何封印大模型的记忆，让它老老实实地表现出“无知”，是目前模拟真实环境的一大难题。

6. 慢到令人发指的运行效率 (Efficiency)
大模型本身的自回归（一个词一个词往外蹦）机制就很慢。而 Agent 在做哪怕一个极其简单的动作前，都需要循环调用好几次大模型（读取记忆 -> 思考规划 -> 执行动作）。这种指数级的时间延迟，让很多实时性要求高的任务目前根本无法落地。
